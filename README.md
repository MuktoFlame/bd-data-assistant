# Bangladesh Data Assistant (AI)

An intelligent LangChain-based AI assistant powered by Google Gemini that answers queries by intelligently routing between three Bangladesh-specific SQLite databases and the open web. It provides instant, accurate data on local hospitals, educational institutions, and restaurants.

---

## 📐 Architecture

```
User Query
    │
    ▼
Main Agent (LangChain create_agent + Gemini)
    │
    ├──▶ InstitutionsDBTool  ──▶ institutions.db  (universities, colleges, govt bodies)
    ├──▶ HospitalsDBTool     ──▶ hospitals.db      (hospitals, beds, type, location)
    ├──▶ RestaurantsDBTool   ──▶ restaurants.db    (cuisine, rating, location)
    └──▶ WebSearchTool       ──▶ Tavily / SerpAPI / DuckDuckGo (general knowledge)
```

### Routing Logic

| Query type | Tool used |
|---|---|
| "How many hospitals in Dhaka?" | `HospitalsDBTool` |
| "Top-rated restaurants in Chittagong?" | `RestaurantsDBTool` |
| "List universities in Sylhet" | `InstitutionsDBTool` |
| "What is the healthcare policy in Bangladesh?" | `WebSearchTool` |
| "What is DGHS?" | `WebSearchTool` |

## ⚙️ Setup & Installation

### 1. Clone the repository

```bash
git clone https://github.com/MuktoFlame/bd-data-assistant.git
cd bd-data-assistant
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Set API keys

Create a `.env` file (or export variables):

```bash
# Required — Google Gemini (free via AI Studio)
# Get key at: https://aistudio.google.com/app/apikey
GOOGLE_API_KEY=your-google-key

# Web search — DuckDuckGo is the free fallback (no key needed)
# TAVILY_API_KEY=your-tavily-key          # optional
# SERPAPI_API_KEY=your-serpapi-key        # optional
```

### 4. Build the databases

Downloads the three HuggingFace datasets and converts them to SQLite:

```bash
python setup_databases.py
```

### 5. Run the agent

```bash
python agent/main_agent.py
```

## 🌐 Web Search Backends

The `WebSearchTool` tries providers in this priority order:

1. **Tavily** — Set `TAVILY_API_KEY` ([get key](https://tavily.com))
2. **SerpAPI** — Set `SERPAPI_API_KEY` ([get key](https://serpapi.com))
3. **DuckDuckGo** — Free fallback, no key needed (`pip install ddgs`)

## 🛠️ How the SQL Generation Works

Each DB tool's `_run()` method:

1. Reads the table schema from SQLite (`PRAGMA` / `sqlite_master`)
2. Sends schema + natural-language query to the LLM with a strict prompt
3. LLM returns a raw SQLite `SELECT` statement
4. Tool executes the SQL and returns formatted results

This approach keeps each tool small and modular while leveraging the LLM's SQL-writing capability for flexible queries.
